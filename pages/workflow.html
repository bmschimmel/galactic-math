Create Branch
info
git pull origin main, then git checkout -b idt-[num]-description
The idt- prefix links the branch to the Linear issue. Never push directly to main — branch protection will reject it.
Build Solution
info
Edit index.html only. No deps, no localStorage, Web Audio for sounds. Commit: IDT-XX message.
The entire game is one file — no build pipeline needed. The IDT-XX prefix in commit messages is parsed by Linear to auto-link commits to the issue.
Test Locally
info
Open index.html directly in a browser — no server needed.
Verify: quiz runs end-to-end, sounds play, keyboard nav works, comet celebration at ≥75%, Hyperspace and Kessel Run modes, all 5 themes, mobile at ~375px.
Open Pull Request
info
Title: IDT-XX description · Body must include Fixes IDT-XX. Claude never merges its own PR.
Linear: auto → In Review
Fixes IDT-XX in the body is parsed by Linear — when the PR merges, Linear automatically closes the issue. Without it, you'd close it manually.
notifies reviewer
Review + Test
info
Human opens branch locally, verifies the checklist, approves or requests changes.
Check for colors outside CSS variables, any localStorage calls, or external scripts. Also: does it feel right for a 6-year-old? If requesting changes, be specific — Claude re-reads the comment.

Feedback Flow
Kids / Parents / Teachers
info
Primary users are kids ages 5–12. Parents and teachers share and assign it.
Keep UI language simple and encouraging — never punishing. The feedback link lives in the footer so kids don't see it as part of the game.
finds a bug or has feedback
pages/feedback.html
info
Feedback form linked from the footer of the main app.
A static page — no server logic here. On submit it POSTs to the Cloudflare Worker, which holds the GitHub API token server-side so it's never exposed in client code.
POST to worker
Cloudflare Worker
info
worker/feedback-worker.js — receives the form, calls GitHub API to create an issue.
Deployed separately via wrangler deploy — not auto-deployed on git push. Observability and invocation logs are on in wrangler.toml for debugging.
